## 一、产品概述

BonNext 是一款面向全球 Minecraft 玩家社区的桌面启动器，以 **ZZZ 风格的 Neo-Tokyo 赛博朋克美学**为核心视觉语言，致力于提供从浏览内容、管理实例、配置环境到启动游戏的全链路流畅体验。

**核心设计理念：**

- **多壳架构（Multi-Shell）**——用户可在运行时切换完全不同的 UI 体验，当前提供赛博朋克风与苹果风两套完整界面
- **安全与性能并重**——Rust 原生后端确保极致性能与内存安全，AES-256 加密保护用户凭证
- **全链路体验**——集成内容浏览、智能安装、崩溃诊断、社交联机于一体，不只是一个"启动按钮"

**支持平台：** Windows（.msi / .exe）、macOS（.dmg，Universal 二进制）、Linux（.AppImage / .deb）

### 2.1 前端架构

- **框架：** React 18 + TypeScript 5.6 + Vite 6
- **状态管理：** React Context + useReducer 模式，6 大核心 Store（认证、配置、实例、通知、主题、下载）
- **样式系统：** CSS Modules + 设计令牌（Design Tokens），支持 Dark / Light / OLED 三种主题
- **国际化：** 内置 i18n 框架，支持中英文
- **路由：** Hash-based 路由，20+ 页面

### 2.2 后端架构

- **语言：** Rust（Edition 2021），异步运行时 Tokio
- **桌面框架：** Tauri v2
- **IPC 命令：** 约 100 个 Tauri 命令覆盖全部功能
- **模块化设计：** 17+ 独立功能模块，职责清晰

## 四、多壳架构（Multi-Shell）

BonNext 最具创新性的架构设计是 **多壳系统**——允许用户在运行时切换完全不同的 UI 体验。每个 Shell 是一个独立的界面包，拥有自己的页面、组件、样式和设计语言，通过 Shell 注册表统一管理，React.lazy() 按需加载。

### 4.1 ZZZ Shell（默认 — 赛博朋克风）

灵感来源于《绝区零》的 Neo-Tokyo 美学，是 BonNext 的标志性视觉体验。

**设计语言：**

- 主色调：#FFE600 荧光黄 + 深色背景
- 标题字体：Bebas Neue | 正文字体：Inter | 数据字体：DM Mono
- 切角设计（Clip-path）：所有卡片和面板采用斜角切割
- 叠加效果：噪点纹理 + 扫描线（CRT 质感）
- 主题：Dark / Light / OLED 三种

**页面（12+ 核心 + 14 设置子页面）：**

| 页面     | 功能                                       |
| -------- | ------------------------------------------ |
| 首页     | 仪表盘：快速启动面板 + 新闻动态 + 动态背景 |
| 商店     | 市场中心：横幅、分类、趋势内容             |
| 内容详情 | 模组/资源包/光影/整合包的完整信息页        |
| 实例管理 | 实例列表：搜索/过滤/分组                   |
| 实例详情 | 模组/世界/日志/配置/快照                   |
| 内容库   | 已安装内容 + 更新检查                      |
| 模组浏览 | 双源浏览器：标签/画廊/列表/分页            |
| 收藏     | 愿望单管理                                 |
| 版本     | 版本浏览器 + 下载器                        |
| 设置     | 14 个设置子页面                            |
| 下载面板 | Steam 风格浮动下载管理器                   |

### 4.2 SwiftUI Shell（苹果风）

遵循 Apple Human Interface Guidelines，采用 Liquid Glass 毛玻璃质感设计，为 macOS 用户提供原生般的操作体验。

**设计语言：**

- 半透明毛玻璃面板 + 圆角卡片
- SF Pro 字体族
- 精细的阴影层次和弹性动画
- Light / Dark 双主题

**页面（13+）：** 首页、实例管理、实例详情、模组浏览、内容详情、AI 聊天、收藏、版本、设置等完整页面集。

### 4.3 Fluent Shell & TV Shell

- **Fluent Shell**：微软 Fluent Design System 风格（规划中）
- **TV Shell**：电视/大屏模式，适合客厅场景（规划中）

### 4.4 可扩展性

Shell 系统支持从外部文件导入自定义 Shell，开发者可创建并分享自己的界面风格。注册表提供动态注册/注销能力，新的 Shell 无需修改核心代码即可集成。

### 5.2 关键架构决策

1. **选择 Tauri v2 而非 Electron**——安装包体积更小（~10MB vs ~150MB）、内存占用更低、安全性更强
2. **选择 Rust 而非 Node.js 后端**——零 GC 停顿、内存安全、与 Tauri 深度集成
3. **多壳架构而非主题切换**——不只是换颜色，而是完全不同的交互模型和设计语言
4. **React Context + useReducer 而非 Redux**——更轻量、更贴近 React 理念、减少依赖
5. **CSS Modules 而非 CSS-in-JS**——零运行时开销、原生 CSS 能力、更好的隔离性
6. **Shell 懒加载**——用户只下载当前使用的 Shell，减少首屏加载时间

## 六、技术亮点

### 6.1 性能

- **并行下载引擎**：多线程并发、指数退避重试、SHA1 实时校验、三级镜像故障转移
- **实例隔离**：共享库通过硬链接节省磁盘空间，隔离目录确保环境纯净
- **懒加载 Shell**：React.lazy() 按需加载，切换 Shell 无需重启
- **持久缓存**：SQLite 缓存数据库，模组指纹识别，减少重复扫描

### 6.2 安全

- **AES-256-GCM 加密**：HKDF 密钥派生的凭证加密存储
- **P2P 加密传输**：ed25519 身份验证 + x25519 密钥交换 + ChaCha20-Poly1305 加密
- **JVM 白名单**：防止恶意启动参数注入
- **输入消毒**：全链路输入数据清洗
- **安全审计日志**：可追溯的安全事件记录

### 6.3 智能化

- **AI 工作流引擎**：编排式多步骤工作流（安装/修复/批量操作）
- **自然语言交互**：用中文描述需求，AI 自动规划并执行
- **崩溃智能诊断**：自动解析崩溃报告，匹配知识库，提供修复方案
- **模组兼容性评分**：AI 分析模组间的潜在冲突

### 6.4 设计

- **ZZZ 赛博朋克美学**：切角卡片、噪点叠加、扫描线、荧光黄主色——形成独特的品牌辨识度
- **Apple HIG 风格**：SwiftUI Shell 提供 macOS 用户熟悉的操作体验
- **Steam 风格下载面板**：浮动式下载管理器，实时进度、速度、剩余时间
- **3D 过渡动画**：Apple Stage Manager 风格的 AI 面板入场动画

## 九、总结

BonNext 是一款具有独特定位的 Minecraft 启动器——它不仅是功能完备的启动工具，更通过 **多壳架构**、**AI 智能助手** 和 **Neo-Tokyo 美学** 三个核心差异化特性，试图重新定义"启动器"的边界。

在三周的高强度开发中，项目从零搭建了完整的 Rust + React 技术栈，实现了认证、下载、启动、实例管理、内容平台、AI 助手、社交联机、安全系统等 8 大功能域，覆盖约 100 个后端命令和 20+ 前端页面。多壳架构的引入使 BonNext 成为一个可扩展的 UI 平台，而非单一界面的工具。

当前 v0.0.5 作为 Pre-Release 版本，核心链路已跑通，两套 UI 风格已可体验。下一阶段将聚焦于体验打磨（对比度、进度反馈、国际化）和功能补全（模组开关、异步 IO 优化），为正式发布做好准备。
