# BonNext Wiki 文件树功能优化设计文档

> **日期**: 2026-05-29  
> **范围**: wiki-v2 文件树导航系统的全面重构  
> **参照竞品**: VitePress, Docusaurus, GitBook

---

## 一、当前问题诊断

### 1.1 Critical 问题

| 问题               | 影响                                              | 竞品表现                         |
| ------------------ | ------------------------------------------------- | -------------------------------- |
| 文件夹标题不可点击 | 用户无法通过点击文件夹展开/折叠，箭头是唯一交互点 | VitePress/GitBook 均支持点击标题 |
| 折叠状态不持久化   | 刷新后所有折叠状态丢失，用户需重新展开            | 全部竞品使用 localStorage 持久化 |
| 搜索仅过滤可见项   | 折叠文件夹内的匹配项无法被搜索到                  | 全部竞品支持全局搜索             |

### 1.2 Major 问题

| 问题                   | 影响                                     | 竞品表现                             |
| ---------------------- | ---------------------------------------- | ------------------------------------ |
| 面包屑中间节点不可点击 | 无法通过面包屑快速跳转上级分类           | VitePress/Docusaurus 面包屑全可交互  |
| 无当前路径自动展开     | 页面加载后当前文件可能隐藏在折叠文件夹中 | 全部竞品自动展开当前路径             |
| 搜索无防抖             | 大数据量时输入卡顿                       | 全部竞品有防抖/节流                  |
| 无文件类型图标         | 无法一眼区分文档类型                     | GitBook 有图标，VitePress 有分组样式 |

### 1.3 Minor 问题

- 移动端无遮罩层
- 无键盘导航支持
- 无滚动位置记忆
- 折叠动画生硬（max-height 跳变）

## 三、优化方案（方案 C：完整增强）

### 3.1 核心功能清单

#### F1: 文件夹可点击交互

- 点击文件夹标题 = 展开/折叠
- 点击箭头 = 展开/折叠（与标题行为一致）
- 双击文件夹标题 = 若该文件夹有索引页，跳转到索引页

#### F2: 状态持久化

- localStorage 键: `bnwiki_nav_state`
- 保存内容: 每个文件夹的折叠状态（以路径为 key）
- 保存滚动位置: `bnwiki_scroll_pos`
- 页面加载时恢复状态

#### F3: 全局搜索（自动展开 + 高亮）

- 搜索时遍历所有节点（包括折叠的）
- 自动展开包含匹配项的父文件夹
- 高亮匹配文本（`<mark>` 标签）
- 150ms 防抖

#### F4: 面包屑可交互

- 中间节点变为链接
- 链接目标为该分类的索引页（或第一个子页面）

#### F5: 当前路径自动展开

- 页面加载时，自动展开当前文件的所有父文件夹
- 当前文件高亮（已有，确保在折叠恢复后仍然正确）

#### F6: 搜索防抖

- 输入防抖 150ms
- 使用 `requestAnimationFrame` 批量 DOM 更新

#### F7: 文件类型图标

- 根据文件路径/分类自动分配图标
- 或使用 sidebar.json 中配置的图标

#### F8: 页面内 TOC（Table of Contents）

- 自动提取页面 H2/H3 标题
- 右侧浮动 TOC 或页面顶部 TOC
- 滚动时高亮当前阅读位置

#### F9: 上一页/下一页分页导航

- 页面底部显示相邻文档链接
- 基于文档树的中序遍历顺序

#### F10: 本地全文搜索索引

- 构建时生成 `search.json`
- 包含所有页面的标题、内容摘要
- 客户端 MiniSearch 实现实时搜索

#### F11: 键盘导航

- `Tab` / `Shift+Tab`: 在可聚焦元素间移动
- `↑` / `↓`: 在导航树中上下移动
- `←` / `→`: 折叠/展开当前文件夹
- `Enter`: 激活当前项
- `Escape`: 关闭移动端侧边栏 / 清除搜索

#### F12: 移动端遮罩层

- 打开侧边栏时显示半透明遮罩
- 点击遮罩关闭侧边栏

#### F13: 平滑折叠动画

- 使用 `grid-template-rows` 动画（比 max-height 更流畅）
- 支持 `prefers-reduced-motion`

### 3.2 配置驱动（sidebar.json）

```json
{
  "sidebar": [
    {
      "type": "group",
      "title": "概述",
      "icon": "📖",
      "collapsible": true,
      "collapsed": false,
      "items": [
        { "type": "link", "title": "项目简介", "href": "/01-概述/01-项目简介" },
        { "type": "link", "title": "设计理念", "href": "/01-概述/02-设计理念" }
      ]
    },
    {
      "type": "group",
      "title": "指南",
      "icon": "🚀",
      "collapsible": true,
      "collapsed": true,
      "items": [
        { "type": "link", "title": "快速开始", "href": "/02-指南/01-快速开始" },
        { "type": "link", "title": "开发环境", "href": "/02-指南/02-开发环境" }
      ]
    }
  ]
}
```

- 若 `sidebar.json` 不存在，回退到文件系统自动扫描
- 构建时校验配置与文件系统一致性

### 3.3 性能优化

- **大数据量**: 文档树 > 50 项时，启用虚拟滚动（仅渲染可视区域节点）
- **搜索索引**: 构建时预生成，避免运行时遍历 DOM
- **DOM 更新**: 使用 `DocumentFragment` 批量更新

## 四、技术架构

```
构建时:
  content/ + sidebar.json ──→ build.js ──→ dist/
                                     ├──→ 静态 HTML 页面
                                     ├──→ search.json (全文索引)
                                     └──→ css/style.css

运行时:
  客户端 JS ──→ localStorage 状态恢复
           ──→ 导航树交互 (折叠/展开/搜索)
           ──→ TOC 高亮
           ──→ 键盘导航
           ──→ 分页导航
```
